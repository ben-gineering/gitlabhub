# Quick Start Guide

## Prerequisites

Before using the script, ensure you have:

1. **GitLab CLI (glab)** installed
2. **GitHub CLI (gh)** installed
3. **Git** installed
4. Both CLIs authenticated

## Quick Setup

### 1. Install and Authenticate

```bash
# Install glab (if needed)
brew install glab                      # macOS
sudo apt install glab                  # Linux

# Install gh (if needed)
brew install gh                        # macOS
sudo apt install gh                    # Linux

# Authenticate with GitHub
gh auth login

# Authenticate with GitLab
glab auth login
```

### 2. Run the Script

```bash
# Download the script
git clone https://github.com/yourusername/gitlabhub.git
cd gitlabhub

# Make executable
chmod +x setup-mirror.sh

# Run with your project name
./setup-mirror.sh my-project
```

### 3. Verify Mirroring

After the script completes:

```bash
# Check remotes
git remote -v

# Should show:
# origin  git@gitlab.com:username/my-project.git (fetch)
# origin  git@gitlab.com:username/my-project.git (push)
# mirror  git@github.com:username/my-project.git (fetch)
# mirror  git@github.com:username/my-project.git (push)

# Check mirror status in GitLab
# Go to: https://gitlab.com/username/my-project/-/settings/repository
# Look for "Mirroring repositories" section
```

## Usage Examples

### Public Project (Default)

```bash
./setup-mirror.sh my-awesome-project
```

Creates:
- `my-awesome-project/` folder
- Public GitLab repository
- Public GitHub repository
- Push mirroring (GitLab â†’ GitHub)

### Private Project

```bash
./setup-mirror.sh --private my-secret-project
```

Creates:
- Private repositories on both platforms
- All other features same as public

### Explicit Public

```bash
./setup-mirror.sh --public my-open-project
```

Same as default, but explicitly public.

## Workflow After Setup

### Make Your First Commit

```bash
cd my-project

# Add some files
echo "# My Project" > README.md
touch .gitignore LICENSE

# Commit and push
git add .
git commit -m "Initial commit with project files"
git push origin main
```

### Verify GitHub Mirror

After pushing to GitLab:

```bash
# Wait ~5 minutes for automatic sync
# Or check status in GitLab UI

# View GitHub repository
gh repo view

# Should show the same commits as GitLab
```

### Ongoing Development

```bash
# Create a feature branch
git checkout -b feature/new-feature

# Make changes
echo "New feature" >> features.txt
git add .
git commit -m "Add new feature"

# Push to GitLab
git push origin feature/new-feature

# GitHub will automatically receive the update
```

## Troubleshooting

### Script Fails with "Command Not Found"

```bash
# Check if tools are installed
which glab
which gh
which git

# If not found, install them (see prerequisites above)
```

### Authentication Errors

```bash
# Re-authenticate both CLIs
gh auth logout && gh auth login
glab auth logout && glab auth login
```

### Repository Already Exists

```bash
# Choose a different name
./setup-mirror.sh my-project-v2

# Or delete existing repositories first
gh repo delete username/repo-name
# (You'll need to confirm with --æ™–è¡Œ-mode-in)
```

### Push Mirror Not Working

1. Check GitLab mirror configuration:
   ```bash
   glab api projects/username%2Frepo/remote_mirrors
   ```

2. Verify GitHub token has `repo` scope

3. Check GitLab logs for errors

4. Manual configuration:
   - Go to GitLab â†’ Project â†’ Settings â†’ Repository
   - Scroll to "Mirroring repositories"
   - Add your GitHub repository URL
   - Select "Push" direction
